Rules:
- Aggregate roots are classes that own a cluster of related objects
- Value objects are immutable and identified by value, not identity (e.g. Money, Address)
- Domain services contain business logic not belonging to a single entity
- Domain events represent something that happened in the domain
- Only include entities with clear business meaning, not technical/infrastructure classes
- Use source_class when you can directly map a domain entity to a class in the summary
- Set confidence < 0.5 for speculative domain concepts
