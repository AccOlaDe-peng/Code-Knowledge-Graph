# 一、产品设计文档（PRD）

---

## 1.1 产品背景

随着项目规模增长，开发者对代码结构理解成本显著上升：

- 依赖关系复杂
- 调用链难以追踪
- 新人上手困难
- 架构风险不可视

CodeGraph 提供：

> 自动拉取代码 → 结构解析 → 构建依赖图 → 可视化展示

帮助团队快速理解系统结构。

---

## 1.2 产品目标（Phase 1）

### 🎯 核心目标

- 支持代码仓库接入
- 自动生成结构依赖图
- 提供可视化与查询能力
- 支持 SaaS 多租户

---

## 1.3 用户角色

| 角色       | 目标             |
| ---------- | ---------------- |
| 开发者     | 快速理解项目结构 |
| 架构师     | 分析模块依赖     |
| 技术负责人 | 识别结构复杂度   |

---

## 1.4 核心功能范围（MVP）

### 1️⃣ 用户系统

- 注册 / 登录
- JWT 鉴权
- 项目隔离

---

### 2️⃣ 项目管理

- 创建项目
- 删除项目
- 查看项目状态
- 查看分析进度

---

### 3️⃣ 仓库接入

支持：

- GitHub OAuth
- GitLab OAuth
- ZIP 上传

能力：

- 选择仓库
- 选择分支
- 自动拉取代码

---

### 4️⃣ 代码结构分析

仅支持：

- JavaScript
- TypeScript

解析内容：

- 文件依赖（import）
- 函数调用
- 类依赖
- 模块归属关系

---

### 5️⃣ 图谱展示

视图类型：

- 文件依赖图
- 函数调用图
- 模块级视图

支持：

- 折叠
- 搜索
- 高亮

---

### 6️⃣ 查询能力

- 查看某文件被谁引用
- 查看某函数调用链
- 查找循环依赖

---

## 1.5 非功能需求

- 单项目 ≤ 10k 文件
- 分析时间 ≤ 5 分钟
- 并发用户 ≥ 100
- 数据隔离
- 支持横向扩展

---

## 1.6 本阶段不做

- AI 语义分析
- 多语言支持
- CI 集成
- 企业私有部署

# 二、技术架构文档

---

## 2.1 总体架构

```
[Frontend]
    ↓
[API Gateway]
    ↓
[Backend Service]
    ↓
[Task Queue]
    ↓
[Analysis Worker]
    ↓
[Graph DB] + [Relational DB]
```

---

## 2.2 技术选型

### 前端

- vue
- Vite
  -TypeScript
  -Pinia
  -Vue Router
  -Element Plus
- Cytoscape.js 或 G6（图可视化）
  推荐图引擎：

Cytoscape（轻量）

## G6（复杂图支持更强）

### 后端

- NestJS
- RESTful API

---

### 代码分析引擎

- Node.js
- ts-morph（AST 解析）

---

### 图数据库

使用：

### Neo4j

原因：

- 成熟稳定
- 支持复杂路径查询
- 易扩展

---

### 业务数据库

- PostgreSQL

存储：

- 用户
- 项目
- 任务
- 配额

---

### 消息队列

- Redis
- BullMQ

---

## 2.3 分析任务流程

```
用户提交分析
 ↓
生成任务记录
 ↓
推入队列
 ↓
Worker 拉取
 ↓
克隆仓库
 ↓
AST 解析
 ↓
构建图数据
 ↓
写入 Neo4j
 ↓
任务完成
```
